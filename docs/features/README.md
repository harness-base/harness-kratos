# 需求包（features）

用户可见的需求 / 行为 / 验收变化，先在这建需求包，再动业务代码（rule-0001）。

- 模板：`templates/feature-package.md`
- 账本：`index.yaml`（登记每个包的 id / 标题 / delivery_status / implementation_allowed）
- 真需求随被管工程进 `projects/` 后填充。
