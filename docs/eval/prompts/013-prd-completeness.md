# eval 013 — PRD/原型 完整性与可观测性（rule-0010）

## 评什么

`prd-elicitation` 产出的 PRD + 原型，够不够"想清楚了、能往下做"——重点是**可观测验收**与**原型真可点通**，而不是一份华丽但无法落地/无法验收的文档。空泛验收（"支持 X / 做完即可"）、范围只写包含、原型是静态图、假设没确认，都要打回。

## 检查清单（逐条查）

- **验收可观测**：每条验收目标是否可观察、可判定？出现"做完了/支持/完善"这类不可验证措辞即 fail。
- **范围闭合**：`包含` 与 `不包含` 是否都写了？只有包含 = fail。
- **四态齐**：每个页面/视图是否覆盖 空/加载/错误/成功 + 关键边界？
- **原型可点通**：读 `prototype/`——主流程交互是否**真接了**（导航/表单/校验/状态切换），还是静态 mockup？只有静态页 = fail。**有现有前端**时，原型内容/风格是否与之一致？
- **假设已确认**：`假设与待定` 是否显式列出且经用户确认，而非静默假设？
- **可追溯**：验收目标 ↔ 页面 ↔（未来）测试 是否有映射？
- **登记一致**：PRD 是否登记进 `docs/prds/index.yaml`、目录与账本一致（机器侧 `prds-audit.sh` 已查，这里抽验）？

## 输出

verdict（pass / fail）+ 命中问题的 `文件:位置` 与具体证据（哪条 AC 不可观测、哪页缺态、原型哪段是静态没接交互）+ 修法建议。
