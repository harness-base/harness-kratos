# eval 013 — 用户故事/PRD/原型 完整性与可观测性（rule-0010 / ADR-0007）

## 评什么

`prd-elicitation` 分阶段产出（用户故事 → PRD →（可选）原型）够不够"想清楚了、能往下做"——重点：**用户故事先行且 approved**、**PRD 与故事对齐且功能点覆盖**、**验收可观测**、**原型（若产出）真可点通**。华丽但无法落地/无法验收、跳过用户故事、功能点不覆盖、原型是静态图、假设没确认，都打回。

## 检查清单（逐条查）

- **用户故事先行**：`user-stories.md` 是独立产物、每条带 `US-NN`、`us_status: approved`？PRD 是否在故事 approved 后产出、与之对齐（不另写需求）？故事缺失 / 未确认即 fail。
- **功能点覆盖（软，目标 100%）**：PRD 有功能点清单 + `US↔FP↔正文` 映射表？每个 US 有 FP 覆盖、每个 FP 追到 US 且正文有描述、无孤儿内容？有缺口须显式说明，无说明的遗漏 = fail。
- **验收可观测**：每条验收目标可观察、可判定？出现"做完了/支持/完善"这类不可验证措辞即 fail。
- **范围闭合**：`包含` 与 `不包含` 都写了？只有包含 = fail。
- **四态齐**：每页覆盖 空/加载/错误/成功 + 关键边界？
- **原型**：若产出——`prototype/` 主流程交互**真接了**（导航/表单/校验/状态切换）、非静态 mockup；有现有前端则内容/风格一致。无界面 / 用户不要而跳过 = n/a（须确认是合理跳过，非偷懒）。
- **旧内容关联**：触及现有系统时，旧关联找齐、旧场景简 / 繁分别处理（复杂的附引用来源）？凭印象臆造旧行为 = fail。
- **假设已确认**：`假设与待定` 显式列出且经用户确认，非静默假设？
- **登记一致**：PRD 登记进 `docs/prds/index.yaml`（条目含 `dir: <id>`）、目录与账本一致；`user-stories.md` 存在性由 `prds-audit` 查（机器侧已查，这里抽验）？

## 输出

verdict（pass / fail）+ 命中问题的 `文件:位置` 与具体证据（哪条 AC 不可观测、哪个 US 没 FP 覆盖、哪页缺态、原型哪段静态没接交互）+ 修法建议。
