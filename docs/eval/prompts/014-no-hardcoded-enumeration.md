# eval 014 — 状态/索引文档不硬编码可自动生成的枚举（rule-0012）

## 评什么
本次若改动了状态/索引类文档（如 `CURRENT_STATUS.md`、各 `README` 职责表），是否把"已有 `*-index` 自动生成权威清单"的信息硬编码成了计数/枚举。

## 通过标准
- 凡有自动生成索引的清单（skill→`.agents/skills/README.md`、规则→`docs/rules/index.yaml` 等），状态文档写"以该索引为准"的指针，不复刻整列清单/计数。
- 举 1–2 个代表例做说明可以。
- 未触碰这类文档 → n/a。

## 判失败
- 在状态文档里硬编码了一份与自动生成索引重复的整列枚举/计数（= 未来漂移源），且无 `--check` 守。
- 发现旧的硬编码枚举已漂移（计数/清单与现状不符）却没顺手改成指针。
