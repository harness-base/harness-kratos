# 测试用例产出（AC/FP → 用例 → 覆盖闭合）

本 skill 管"**产出测试用例 + 用例对需求的覆盖**"——把需求侧的验收点（AC）/ 功能点（FP）系统转成测试用例，并保证覆盖不漏。**只管"用例齐不齐 / 覆盖全不全"，不碰"过没过"**（真跑 / 执行结果是后面单独一段机制）。这里的"覆盖率"指**用例对需求的覆盖**，不是跑出来的代码覆盖率。**独立** skill：AC/FP 当通用输入，与上游 `prd-elicitation` / 下游 `dev` 的衔接后定。流程依据见 ADR-0008。

## 何时用 / 何时不用
- 用：要把验收点 / 功能点转成测试用例；要管"用例对需求覆盖全不全"；评审某需求的用例齐不齐。
- 不用：要"跑用例 / 看过没过"（执行结果，本 skill 不碰）；纯 bug 修复走 `dev`（改 bug 子模式）；纯控制面改动。

## 覆盖模型（软+硬分层，按"机器查不查得了"切）
- **硬闸**（`scripts/test-cases-audit.sh` → `make verify` 红）：可机检的**存在性 / 完整性**——① 每条 AC、每个 FP 都被 ≥1 条用例 `covers:` 覆盖（无遗漏）；② `covers:` 引用的 AC/FP 都已声明（无悬空）；③ 目录 ↔ `index.yaml` 登记一致。
- **软**（本 skill 对抗评审 + 收尾 eval 015）：机器查不了的**质量**——用例是否真覆盖该 AC/FP 的语义与边界、正常 / 边界 / 异常齐不齐、有没有为凑覆盖率的空壳用例。
- **`covers:` 是覆盖关系的唯一真相源**：人只维护各用例的 `covers:`，不另存手维护映射表（防漂，rule-0012）。

## 工作流（每步可多轮 + 用户确认门）
1. **取 AC/FP**：从需求来源取验收点 + 功能点——用户口述 / 指 `user-stories.md` 的验收点 / 指 feature 包 AC，**通用不绑定来源**。誊录进 `docs/test-cases/<id>/test-cases.md` 的 `## 验收点 AC`（`- AC-n：…`）+ `## 功能点 FP`（`- FP-n：…`），每条稳定 id。信息不足 → 去查事实源或问用户，**不静默假设**（同 rule-0008）。**段标题承重**：声明必须落在以「验收点」/「功能点」起始的 `## ` 段内、`covers:` 必须在以「用例」/「测试用例」起始的 `## ` 段内——改标题名 / 声明落段外会被硬闸判红或漏算（套 `templates/test-case.md` 照写即可）。
2. **逐 AC/FP 设计用例**：套 `templates/test-case.md`，每条 `### TC-n` 标类型（正常 / 边界 / 异常）+ 前置 + 步骤 + 可观察预期 + `covers: AC-x, FP-y`（**`covers:` 写单行，勿折行**）。每条 AC/FP 至少有正常用例 + 必要的边界 / 异常。
3. **覆盖闭合自检（硬）**：跑 `bash scripts/test-cases-audit.sh`——每条 AC/FP 都被覆盖、无悬空、账本一致；缺则补用例 / 修 `covers:`。
4. **对抗式完整性自查（软）**：逐条核——用例真覆盖该 AC/FP 语义？边界 / 异常齐？有无空壳凑数用例？必要时多 agent 对抗证伪，审 → 修 → 再审到没问题。
5. **登记 + 收尾**：`docs/test-cases/index.yaml` 登记（`id`/`dir`/`title`/`status`/`source`）；`make verify` 绿（硬闸过）；L2+ 跑收尾 eval **考题 015**（rule-0005）。

## 硬规则
- **不碰执行结果**：只产出用例 + 管覆盖，"过没过"另起。
- **AC + FP 双轴全覆盖**：每条 AC、每个 FP 都被 ≥1 用例 `covers:` 覆盖（硬闸机检）。
- **`covers:` 是唯一真相源**：不另存手维护映射表。
- 用例覆盖**正常 / 边界 / 异常**，不只正常路径凑数。
- 产物**独立存放**于 `docs/test-cases/<id>/`，登记不漂移。

## 门禁（rule-0014 / 考题 015）
- 产出测试用例须满足 **rule-0014**（测试用例产出标准）。
- 结构 / 覆盖闭合由 `scripts/test-cases-audit.sh` 机器校验（+ 守护测试 `test-cases-audit.test.sh`），进 `make verify`。
- 收尾跑 eval **考题 015**（用例覆盖完整性与质量）。
- **不强制测试用例必须存在**（松耦合）——门禁只在产出时适用。

## 演进（rule-0007）
覆盖模型 / 模板 / 硬闸字段变化时回顾本 skill（连同 `templates/test-case.md`、`scripts/test-cases-audit.sh` 同源）；改完同步 `version` / `last_reviewed`，跑 `bash scripts/skills-index.sh`。
