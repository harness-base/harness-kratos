# Database Migrations

Migration SQL is intended to be generated by Atlas from the ent schema. In
practice the atlas ent loader rejects the schema because it lives under a Go
`internal` package, so the initial migration (`20260602000000_initial.sql`) is
**maintained by hand** to match the ent schema and applied to the sandbox via
initdb (`deploy/sandbox/initdb`). See that SQL file's header for how to switch
to atlas generation.

## Generating migrations

Atlas requires a running Docker daemon for the hc-dev database:

```sh
make -C projects/kratos-base migrate
```

This runs:

```sh
atlas migrate diff initial \
  --dir "file://deploy/migrations" \
  --to "ent://./app/demo/internal/data/ent/schema" \
  --dev-url "docker://postgres/16-alpine/dev"
```

(The `migrate` target tolerates failure: if the atlas loader hits the `internal`
restriction or Docker is unavailable, it warns and exits 0 — see the Makefile.)

Migration generation is intentionally excluded from `make verify` because it
requires Docker.
